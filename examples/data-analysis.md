# 数据分析流程优化示例

本示例展示如何使用 Multi-Agent Orchestrator 并行优化数据分析流程。

## 场景

数据分析 pipeline 运行缓慢，需要从 4 个方向并行优化：
- 查询优化 (SQL/索引)
- 缓存策略 (Redis/Memcached)
- 并行计算 (多线程/分布式)
- 存储优化 (分区/压缩)

## 实施路线图

```
Month 1:
├─ Week 1: 查询优化 (风险低，见效快)
│   └─ 添加索引、重写 SQL
├─ Week 2: 存储优化 (基础建设)
│   └─ 分区、格式转换
└─ Week 3-4: 缓存策略 (依赖存储优化)
    └─ Redis 部署、缓存逻辑实现

Month 2:
└─ Week 1-2: 并行计算 (最大改进)
    └─ Dask 集群搭建、任务并行化

Month 3:
└─ 监控、调优、文档完善
```

## 分步命令

```bash
# 1. 诊断分析
/mao analyze "优化数据分析流程，当前4小时处理10GB"

# 2. 指定优化方向
/mao decompose \
  --modules 4 \
  --names "查询优化,缓存策略,并行计算,存储优化" \
  --constraints "independent" \
  --deliverable "optimization-proposal"

# 3. 启动
/mao launch

# 4. 检查冲突
/mao detect-conflicts

# 5. 协调方案
/mao coordinate --strategy phased

# 6. 生成最终方案
/mao finalize --include-roi --include-timeline
```
